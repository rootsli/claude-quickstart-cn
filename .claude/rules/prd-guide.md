# PRD 编写指南

将创意想法转化为 PRD（Product Requirements Document，产品需求文档）的标准流程。
只要说出「想做一款这样的产品」，就可以按照本指南进行系统化梳理。

## 阶段 1：想法具象化

请回答以下 4 个核心必答问题。

| 序号 | 问题 | 示例 |
|---|------|------|
| 1 | 要做什么产品？	 | 基于 AI 的运动计划推荐 App |
| 2 | 三大核心功能是什么？ | 体能测评、运动计划生成、训练进度追踪 |
| 3 | 面向谁？ | 运动初学者（20-30 岁） |
| 4 | 解决什么痛点？ | 不用私人教练，也能拥有定制化运动方案 |

可选补充问题（用于进一步细化）：
- 和同类产品的差异化优势是什么？
- 盈利模式是什么？（订阅 / 广告 / 一次性付费）
- 预计上线时间是什么时候？

## 阶段 2：商业可行性评估

### 立项判定 Go/No-Go

共 5 项维度，每项按 1–5 分打分评估。

| 评估项 | 评判标准 | 得分 |
|------|------|------|
| 市场规模 | 潜在用户是否足够多 | /5 |
| 成长性 | 市场是否处于增长趋势 | /5 |
| 竞争强度 | 现有竞品是否偏少或实力较弱 | /5 |
| 差异化 | 是否具备清晰的特色优势 | /5 |
| 盈利模式 | 是否具备可盈利的商业模式 | /5 |

- 总分 20 分及以上：**立项推进（Go）**
- 总分 15–19 分：**有条件立项（补齐短板后推进）**
- 总分 14 分及以下：**放弃立项（考虑转型或终止）**

## 阶段 4：生成 PRD 文档

按以下 10 个标准章节编写完整 PRD。

| 序号 | 章节 | 内容说明 |
|---|------|------|
| 1 | 产品概述 |一句话介绍产品 + 核心价值 |
| 2 | 开发背景 | 为什么需要这款产品、痛点定义 |
| 3 | 商业分析 | 市场规模、竞品分析、盈利模式 |
| 4 | 核心功能 | 功能列表 + 优先级划分 (P0/P1/P2) |
| 5 | 非功能性需求 | 性能、安全、可用性相关标准 |
| 6 | 业务范围 | MVP 最小范围 vs 完整版范围、包含 / 不包含功能 |
| 7 | 约束条件 | 技术限制、时间限制、预算限制 |
| 8 | 技术调研 | 整理阶段 3 的调研结果 |
| 9 | 设计参考 | 参考设计、UI 截图素材 |
| 10 | 技术栈 | 最终选定技术栈及选型理由 |

### 功能优先级定义
- **P0（必需）：缺少该功能则产品无法上线运行
- **P1（重要）：对用户体验影响极大的核心功能
- **P2（可选）：有则更好，可后续迭代补充的功能

## 阶段 5：后续行动选择

PRD 编写完成后可选择以下操作：
1. **开始开发**：基于 PRD 制定开发实施计划
2. **文档修改**：补充、调整 PRD 内容
3. **保存文档**：将文档存入项目目录 `docs/PRD.md`

### PRD 标准存放路径
```
项目根目录/
├── docs/
│   └── PRD.md          <- PRD 文档
├── CLAUDE.md
└── ...
```
