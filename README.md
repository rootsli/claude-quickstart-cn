# Claude 中文版入门套件
 
Claude 中文版入门配置套件，包含Claude全局配置模板、项目级配置模板、.claude配置模板（hooks、rules、skills等），让你从0开始快速配置项目的Claude规则。

## 安装方法

1. **全局规则**：将 `global-CLAUDE.md` 内容复制到全局配置目录并重命名为 `~/.claude/CLAUDE.md`【可选】
2. **项目规则**：将 整个`.claude/` 文件夹完整复制到项目根目录
3. **项目规则**：将 `CLAUDE.md` 复制到项目根目录
4. [**安装Superpowers**](https://github.com/obra/superpowers)：Superpowers包含了一系列技能和指令的集合(例如：头脑风暴、代码审查、TDD、debugging等技能)，能极大提升Claude Code的功能和效率【推荐】
5. 安装完成

```
项目根目录/
├── global-CLAUDE.md                   <- 全局配置
├── CLAUDE.md                          <- 项目级配置
├── .claude/                           <- 完整复制此文件夹
│   ├── settings.json                  <- 钩子配置（自动PRD检测）
│   ├── hooks/
│   │   └── prd-detector.py            <- PRD关键字检测钩子
│   ├── skills/
│   │   └── debug-helper/
│   │   │   └── SKILL.md               <- /debug-helper 技能
│   │   └── git-commit/
│   │   │   └── SKILL.md               <- /git-commit 技能
│   │   └── prd-create/
│   │       └── SKILL.md               <- /prd-create 技能
│   └── rules/
│       ├── coding-style.md            <- 编码风格规范
│       ├── date-rules.md              <- 日期计算规则
│       ├── execution-rules.md         <- 执行完整性规则
│       ├── prd-guide.md               <- PRD编写参考指南
│       ├── project-rules.md           <- 开发流程规范
│       └── security-rules.md          <- 安全规则
├── src/
└── ...
```

6. 根据自己的项目修改 CLAUDE.md 里的"技术栈"部分
7. 运行 Claude Code 后会自动应用规则

## 各文件作用说明

| 文件 | 作用 | 说明 |
|------|------|------|
| `CLAUDE.md` | 项目全局配置 | 包含项目语言、技术栈、质量、安全、架构等8大板块规则 |
| `settings.json` | 钩子配置 | 配置 PRD 关键词自动检测功能 |
| `prd-detector.py` | PRD检测钩子 | 识别用户输入中的 PRD 相关关键词，并给出使用指引 |
| `SKILL.md` | PRD生成技能 | 输入 `/prd-create` 即可触发五步 PRD 文档编写流程 |
| `coding-style.md` | 代码编写规范 | 命名规范、常量定义、类型约束、错误处理等 8 项规范 |
| `project-rules.md` | 开发流程规范 | TDD、事前风险校验、提交规范、错误处理等 10 项规则 |
| `date-rules.md` | 日期计算规则 | 日期计算必须通过代码实现，禁止依靠大模型脑补推理，共 4 条规则 |
| `execution-rules.md` | 执行完整性规则 | 修改后校验、全局检索、变体检索等 4 条执行规范 |
| `security-rules.md` | 安全规则 | 身份认证与授权、OWASP 十大漏洞、密钥管理等 6 大板块 |
| `prd-guide.md` | PRD编写参考 | 将产品想法落地为标准 PRD 文档的五步骤参考指南 |

## PRD 产品需求文档功能使用方法

直接输入「我想做一款这样的 APP」这类话术，系统会自动引导你进入 PRD 编写流程。

### 方法1：自动检测（钩子触发）

像日常对话一样直接输入即可：
- "帮我做一款运动推荐 APP"
- "帮我写一份需求文档"
- "给我生成一份服务企划案"

检测到 PRD 相关关键词后，会自动提示你使用 `/prd-create` 命令。

### 方法2：手动调用（能力指令）

在 Claude Code 中直接输入 `/prd-create`， 立即启动五步标准 PRD 编写流程。

### 五阶段流程

| 阶段 | 主题 | 具体作用 |
|------|------|--------|
| 阶段 1 | 想法具象化 | 通过 4 个核心问题，把模糊想法梳理成明确需求 |
| 阶段 2 | 商业可行性评估 | 5 项评分做立项判断 + 拆解 3 家竞品分析 |
| 阶段 3 | 技术调研 | 确定技术栈 + 收集参考资料与实现方案 |
| 阶段 4 | 生成 PRD 文档 | 按 10 个标准板块自动生成完整 PRD 文档 |
| 阶段 5 | 后续操作 | 选择保存至`docs/PRD.md`、修改文档、直接开始开发 |

## 钩子(Hook)系统说明

### 什么是钩子？
钩子是在 Claude Code 中触发特定事件时自动执行的脚本。
本项目包含在**用户发送消息时**自动检测 PRD 相关关键词的钩子。

### 工作原理
1. 你向 Claude Code 发送消息
2. `prd-detector.py` 在消息中查找 PRD 关键词
3. 匹配到关键词，自动提示：推荐使用 `/prd-create`
4. 无匹配关键词则静默运行，不影响正常对话

### 自定义关键词
打开文件 `.claude/hooks/prd-detector.py`， 自行修改 `正向关键词列表（POSITIVE_KEYWORDS）` 和 `反向关键词列表（NEGATIVE_KEYWORDS）` 即可。

## 这套配置解决的 5 大开发痛点

| 痛点场景 | 产生原因 | 配置解决方案 | 进阶学习优化方向 |
|------|------|----------------|---------------|
| 改一处、崩多处 | 相同配置散落多处 | 遵循 SSOT 单一数据源原则，仅在一处定义 | 自动校验，杜绝遗漏修改 |
| AI 编码风格每次都不一样 | 无统一规则，AI 随机判断 | CLAUDE.md 提供统一强制规范 | 可按项目定制精细化规则 |
| 同类错误反复出现 | AI 记不住过往失误 | 通过错误处理、安全规则提前规避 | 自动沉淀错误知识库 |
| 代码越来越臃肿混乱 | 文件过大，职责混杂 | 遵循 SRP 单一职责 + 文件行数限制 | 支持自动重构、代码拆分工具 |
| 不知道从哪开始修改维护 | 看不清项目整体结构 | 强制遵循修改前校验、TDD 流程规范 | AI 自动项目结构分析 + 高级工作流 |

## 配置生效校验方法
1. 在 Claude Code 中输入：`告诉我这个项目的所有规则`
2. 确认回复为简体中文
3. 检查是否能准确识别项目技术栈（Next.js、TypeScript 等）
4. 输入：`写代码需要遵守哪些规范？`，若提到 SSOT、SRP 等原则，代表配置已正常生效

## 常见问题(FAQ)

**Q: 需要怎么修改技术栈？**
A: 根据自己的项目修改CLAUDE.md的"技术栈"部分。可以使用React、Vue、Python等任何技术。

**Q: Cursor 编辑器能用吗？**
A: Cursor使用`.cursorrules`文件。将`CLAUDE.md`内容复制到`.cursorrules`中可获得类似效果。

**Q: 配置后感觉没变化？**
A: 重启 Claude Code，再输入「告诉我项目规则」。若仍无识别，检查文件夹路径是否放置正确。

**Q: `/prd-create` 命令没反应？**
A: 请检查 `.claude/skills/prd-create/SKILL.md` 文件是否存在。重启Claude Code后会识别技能。技能仅在Claude Code中工作（不支持Cursor等其他编辑器）。

**Q: 钩子不生效怎么办？**
A: 按以下顺序检查：
1. 检查 `.claude/settings.json` 文件是否存在
2. 检查是否安装了Python 3 (`python3 --version`)
3. 执行测试命令: `echo '{"prompt":"想做应用"}' | python3 .claude/hooks/prd-detector.py`
4. 有输出则正常，无输出则关键字不匹配

**Q: 想关闭钩子**
A: 从 `.claude/settings.json` 中删除 `"hooks"` 部分即可。规则文件(rules/)和技能(/prd-create)无需钩子也能独立运行。

**Q: 测试通过了但仍然出现bug**
A: 测试只会覆盖你编写的场景，需补充空值、网络中断、海量数据等边界异常场景。`project-rules.md` 中的事前风险校验规则，就是用来规避这类遗漏问题。

## 本套件的局限性

这套属于**静态规则基础配置**，不包含以下高阶能力：

- **即便违反规则也不会弹窗警告** -- AI无视规则编写代码时不会自动检测或回滚
- **AI 写出危险操作（删文件、强制推送等）无法自动回滚和拦截** -- 没有在执行危险命令（文件删除、强制推送等）前自动阻止的机制
- **不支持一行命令自动化复杂重复工作** -- 没有自动化组件创建、页面配置等重复任务的工作流
- **无法组建多 AI 分工协作** -- 无法组成分别负责设计、开发、验证的AI团队
